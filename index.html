What CraigClaw Is

CraigClaw is a personal intelligence platform designed to continuously ingest, filter, and synthesize information into a concise daily brief for fast executive decision support.

Signal Selection

Surfaces highest-impact signals instead of broad information dumps.

Context Layer

Explains why each signal matters and how it relates to prior patterns.

Judgment Layer

Applies verification, prioritization, and synthesis before delivery.

Daily Output

Publishes “What Actually Matters” with clear actions and watch items.

How It Is Built

Core Stack

  • Framework: OpenClaw agent orchestration and command layer
  • Host: Dedicated Mac mini for persistent local runtime
  • Scheduling: cron + LaunchAgent

Reasoning Stack

  • Local models: Ollama / Qwen 2.5 for extraction and preprocessing
  • Memory: local embeddings for continuity
  • Cloud judgment: GPT-5.4 via Codex OAuth

Interfaces

  • Control: Telegram
  • Delivery: Email
  • Command: /brief for on-demand output

Architecture Flow

  1. Inputs (Gmail first, RSS next)
  2. Ellie ingests and filters signals
  3. Local models produce structured extraction
  4. Structured signal candidates are created
  5. Fermi applies judgment, verification, and synthesis
  6. “What Actually Matters” is generated
  7. Brief is delivered through Telegram and email

Multi-Agent Ecosystem

CraigClaw is a coordinated agent ecosystem with explicit role boundaries rather than a single assistant.

Active

  • Ellie — Signal Engine: ingestion, deterministic filtering, structured candidates
  • Fermi — Orchestrator & Judgment: prioritization, ranking, verification, synthesis
  • Memory Layer: local embeddings for pattern continuity across days

Planned / Emerging

  • Cato: communication drafting and structured outputs
  • Simons: domain and market intelligence
  • Woz: execution and simplification layer
  • Leo: coordination across agents and workflows
